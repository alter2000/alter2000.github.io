Enter Configuration as code.

Always write everything down

And make it part of the delivery process; don’t slap it in a dead wiki.

The third most common problem in all businesses (after disgruntled employees and inadequate testing of critical elements) is incomplete testing of the whole production. This involves the software built, the environment it is built, the delivery pipeline, and the environment it runs on. The optimal solution for this would be a tool that allows anyone to spin up a miniature production environment and immediately start hacking on it.

All software is one big fancy graph

/nix/store/

Entire system built and linked in one path; parts enabled and disabled via symlinks.

/nix/store/s38iyac9x9piy587ha09qrghha19r4q2-firefox-73.0.1
^---------^^------------------------------^ ^------------^
↪| base path                                 ↪| package name
            ↪| node hash based on contents
             | (matching per derivation in all
             | platforms/environments)
           ^----------------------------------------------^
            ↪| everything here is read-only

Nix: Derivations (DRVs)

$ cat /nix/store/dhmin7wq99aw9f59jm41varj0753va9b-hello-2.10.drv
Derive([("out","/nix/store/4w99qz14nsahk0s798a5rw5l7qk1zwwf-hello-2.10","","")],[("/nix/store/h2la7fpk93khhsabc1z7arm0wbj5aavm-stdenv-linux.drv",["out"]),<...>],<...>)

Every DRV lists all the direct dependencies of a node, and the node hash is based on said DRV, which means once a dependency changes, the DRV and path also change.

Almost every node is created via a DRV (DRVs are nodes themselves).

Easy sandboxing via DRVs

DRV takeaways

  • There is nothing implicit, therefore either everything runs everywhere or nowhere.
  • All software can be a binary package or built from scratch via DRVs and it all behaves the same.

Examples

Per-project development environments

{ nps ? <nixpkgs> }:
with import nps {};

let
  jekyll_env = bundlerEnv {
    name = "jekyll_env";
    ruby = ruby_2_6;
    gemfile = ./Gemfile;
    lockfile = ./Gemfile.lock;
    gemset = ./gemset.nix;
  };
in
  stdenv.mkDerivation rec {
    name = "jekyll_env";
    buildInputs = [ jekyll_env ];

    shellHook = ''
      exec ${jekyll_env}/bin/jekyll serve --watch
    '';
  }
